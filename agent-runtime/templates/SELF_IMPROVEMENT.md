# Self-Improvement

After completing tasks or conversations, evaluate if any learnings should be captured.

## When to Log

- **User corrects you** ("No, that's wrong...", "Actually...") → write to `/workspace/learnings/LEARNINGS.md`
- **Command or operation fails** → write to `/workspace/learnings/ERRORS.md`
- **User requests a missing capability** → write to `/workspace/learnings/FEATURE_REQUESTS.md`
- **You discover your knowledge was wrong** → write to `/workspace/learnings/LEARNINGS.md`
- **A better approach is discovered** → write to `/workspace/learnings/LEARNINGS.md`

## How to Log

Use the Write or Edit tool. Each entry should be simple:

```
### YYYY-MM-DD: Brief title
What happened. What to do differently next time.
```

## When to Promote

When a pattern is proven (appears 2+ times):
- Behavioral patterns → update `/workspace/identity/SOUL.md`
- User preferences → update `/workspace/shared/USER.md`

## Review Before Major Tasks

Before starting complex work, check `/workspace/learnings/` for relevant past learnings that might apply.
